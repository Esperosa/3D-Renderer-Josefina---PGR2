Proč raster

Je rychlý pro viewport, selekci, stylizované renderery a base pass pro post efekty. Zároveň dává G-buffer data, která se používají pro dither, temporal noise a hex mosaic.

Shader pipeline

PhongShader: ambient, diffuse, specular

Role shaderu

PhongShader je CPU shader pro raster mód. Před rendrem si nacachuje aktivní světla a při fragmentu skládá ambient, Lambert diffuse a Blinn-Phong specular pro directional i point/spot lights.

Proč právě Phong v rasteru

Je výrazně levnější než full path tracing, ale pořád čitelně ukazuje tvar, světla, spekulární highlighty a materiálový rozdíl. Hodí se pro real-time viewport.

Render styl

DITHERING: stylizace z luminance a G-bufferu

Blue noise

Procedurální blue-noise threshold mapa rozbíjí pravidelné vzory. Výsledek působí filmověji než čistý Bayer pattern.

Pattern

Bayer 4×4 matice dává kontrolovaný retro vzhled a levnou kvantizaci tónů.

ASCII

Obraz se řeže na buňky; pro každou se hledá glyph, jehož maska nejlépe odpovídá lokální luminanci.

DitherRenderer nejdřív renderuje základní raster do G-bufferu a referenční unlit detail pass. Pak připraví luminanci, aplikuje kontrast, light assist a zvolenou kvantizační strategii.

Architektura

Ray tracing kód

BVH akcelerace průsečíků

Proč BVH

Bez akcelerace by každý paprsek testoval všechny trojúhelníky. BVH rozdělí geometrii do hierarchie AABB uzlů, takže paprsek přeskočí celé oblasti, které nemůže zasáhnout.

public boolean intersect(Ray ray, HitRecord hit) {
    stack.clear();
    stack.push(root);
    while (!stack.isEmpty()) {
        BVHNode node = stack.pop();
        if (!node.bounds.intersects(ray, hit.t)) {
            continue;
        }
        if (node.isLeaf()) {
            testTriangles(ray, hit, node.first, node.count);
        } else {
            stack.push(node.left);
            stack.push(node.right);
        }
    }
    return hit.hit;
}

Preview path

  • může preferovat dominantní contribution,
  • má agresivnější termination pro interaktivitu.

  • plnější BRDF/BSDF transport,
  • lepší direct lighting a MIS,
  • méně heuristických zkratek,
  • použití zejména pro finální path tracing output.

Path kód

Bounce loop a volba lobe materiálu

for (int bounce = 0; bounce < effectiveMaxBounces; bounce++) {
    Hit hit = bvh.intersect(ray);
    if (!hit.hit) {
        radiance += throughput * sampleEnvironment(ray);
        break;
    }

    Surface surface = evaluateMaterial(hit);
    radiance += throughput * surface.emission;
    radiance += throughput * sampleDirectLighting(surface, hit);

    Lobe lobe = chooseLobe(surface, sampler.next1D());
    ray = sampleNextRay(hit.position, surface, lobe, sampler);
    throughput *= lobe.weight / lobe.pdf;

    if (bounce >= 2 && russianRoulette(throughput, sampler)) {
        break;
    }
}

Materiál není jedna jednoduchá barva. Má difuzní, kovovou, skleněnou, clearcoat nebo volume složku. Volba lobe dělí pravděpodobnost mezi tyto fyzikální možnosti.

  • Render styly sdílí scénu a materiály, ale mají vlastní pipeline.
  • Material graph vytváří jednotný autoritativní popis materiálu.
  • Exportní pipeline je oddělená od viewportu a umí zapisovat metadata.
Shrnutí

Jedna věta pro každou hlavní část

Engine

Editor drží scénu, input, kamery, physics a UI ve smyčce, která umí průběžně renderovat viewport i obsluhovat finální output job.

Render

Renderovací architektura sjednocuje raster, stylized post procesy, ray tracing a path tracing přes společný Renderer kontrakt.

Shader

Shader logika je CPU implementace: od fragment Phong shaderu přes G-buffer stylizace až po BSDF/path transport v ray/path rendererech.

Materiály

Node graph je zdroj pravdy a kompatibilní PhongMaterial umožňuje, aby starší i novější renderery čerpaly ze stejného materiálu.

Export

Output pipeline vytváří samostatný render job, renderuje do interního framebufferu a zapisuje still, sekvence, GIF, AVI i session artefakty.

Obhajoba

Nejlepší způsob prezentace je ukázat, že každý render styl není jen filtr, ale separátní rozhodnutí v pipeline s jasným důvodem, vstupy a výstupy.
